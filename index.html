Firebase Push Notifications Guide for React Native (Android)

This guide provides a comprehensive, step-by-step process to implement Firebase Push Notifications in a React Native mobile app for Android, styled in a professional gradient theme inspired by modern AI development tools.

Step 1: Set Up Firebase Project

  • Create a Firebase project in the Firebase Console.
  • Add your Android app by providing the applicationId (from your android/app/build.gradle) and app name.
  • Download the google-services.json file from the Firebase Console.
  • Place the google-services.json file in the android/app/ directory of your React Native project.
  • Install the required Firebase packages with matching versions:
npm install @react-native-firebase/app@^22.2.0 @react-native-firebase/messaging@^22.2.0

üí° Pro Tip: Ensure both @react-native-firebase/app and @react-native-firebase/messaging have the same version (e.g., ^22.2.0) to avoid compatibility issues and ensure smooth integration.

Step 2: Test Push Notifications

Test your Firebase Cloud Messaging (FCM) setup using a tool like FCM Tester. You'll need the FCM token, project ID, and server access token to validate your configuration.

Step 2.1: Get FCM Token

Create a utility function to retrieve the FCM token, handling Android 13+ notification permissions with graceful error handling.

// utils/fcmService.js
import messaging from '@react-native-firebase/messaging';
import { Platform, PermissionsAndroid } from 'react-native';

export async function getFcmToken() {
  try {
    // üöÄ Android 13+ requires POST_NOTIFICATIONS runtime permission
    if (Platform.OS === 'android' && Platform.Version >= 33) {
      const granted = await PermissionsAndroid.request(
        PermissionsAndroid.PERMISSIONS.POST_NOTIFICATIONS
      );
      if (granted !== PermissionsAndroid.RESULTS.GRANTED) {
        console.log('‚ùå Notification permission denied');
        return null;
      }
    }

    // üöÄ Get FCM token
    const token = await messaging().getToken();
    console.log('‚úÖ FCM Token:', token);
    return token;
  } catch (error) {
    console.error('üî• Error getting FCM token:', error);
    return null;
  }
}

Step 2.2: Configure Server Access Token

To send notifications programmatically, obtain a server access token using a Node.js application and the services-account.json file for secure authentication.

  • In the Firebase Console, navigate to Project Settings > Service Accounts.
  • Click Generate New Private Key to download the services-account.json file.
  • Create a Node.js script to generate the access token:
// getAccessToken.js
const { GoogleAuth } = require('google-auth-library');

async function getAccessToken() {
  const auth = new GoogleAuth({
    keyFile: 'services-account.json', // Path to your downloaded JSON
    scopes: ['https://www.googleapis.com/auth/firebase.messaging'],
  });

  const client = await auth.getClient();
  const token = await client.getAccessToken();
  console.log('‚úÖ Access Token:', token.token);
  return token.token;
}

getAccessToken();
  • Install the required Node.js package:
npm install google-auth-library
  • Run the script to obtain the access token.
  • Use the access token and project ID (e.g., project-name-78d63) in the FCM Tester tool to send test notifications.
